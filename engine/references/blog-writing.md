# Blog Writing Reference (Shared Engine)

SEO structure, AEO rules, and post templates. Client-specific local context — cities,
landmarks, employers — lives in [client-profile.md](../client-profile.md) §2. Session model
is §3, credential is §4.

**Read voice-rules.md and voice-persona.md first.** This file is structure and SEO only.

---

## SEO Structure

### Title
- Include the primary keyword AND a city name from the profile's markets list
- Under 65 characters where possible
- Audience-specific and location-specific (see Content Framing Rules in SKILL.md)
- No clickbait — the title describes exactly what the post delivers

### Meta description
- 150–160 characters, includes primary keyword and city
- Ends with an implicit or explicit CTA; written as a sentence, not a keyword list

### URL slug
- Lowercase, hyphens only, no special characters
- Include city name and topic: `studio-headshots-[city]-executives`
- Under 75 characters

### Keyword placement
Primary keyword must appear in: the title (H1), the first 100 words, at least one H2, and the
meta description. Use semantic variants in the body ("professional headshots," "business
portraits," "headshot photographer [city]," "[metro] headshots").

## AEO Structure (Answer Engine Optimization)

Optimized to be surfaced by AI answer engines (ChatGPT, Perplexity, Google SGE, Siri, Alexa)
in addition to traditional search.

1. The intro (first 100–150 words) must be self-contained and answer the core question the
   title poses. If someone asked a voice assistant the title as a question, the intro is the answer.
2. H2 headers written as questions or clear statements matching natural-language queries:
   "Why [City] Professionals Choose Studio Headshots" — not just "Studio Headshots."
3. A "## Common Questions" block (count per the canonical table in SKILL.md; label it that,
   not "FAQ").
4. "Answer boxes" within the body — 3–5 sentence paragraphs that stand alone as a direct answer
   to a sub-question. Not labeled — just written to be self-contained.

## Local Context

Ground posts in specific geography from client-profile.md §2. Each service-area city there has
a line of local texture — a landmark, major employer, or market character. Use those naturally
for authenticity; never force a location reference. Prefer a specific suburb over a generic
"[metro] area" when it adds authenticity. Acknowledge that clients come from across the county
and metro. Respect the session model in §3 (studio, on-location, or both — and what's never depicted).

## Post Structure

### Standard / cluster post (600–1,000 words)
```
[H1: Title with City Name and Audience]
[Intro — 75–125 words. Hook, then answer the core question. AEO-ready.]
[H2: First section — natural-language question or statement]
[Body — 100–200 words]
[H2: Second section]  [Body]
[H2: Third section]   [Body]
[Optional H2: Fourth section]
[Closing — 75–125 words. One takeaway, then CTA.]
```

### Pillar / guide post (1,500–2,500 words)
Same shape, more sections and depth, plus a table of contents and a "## Common Questions"
block. (Canonical word/FAQ counts live in SKILL.md — that table wins over any number here.)

### The intro
Do not start with "If you're a professional in [city] looking to update your headshot..." —
that's a robot sentence. Start with a specific scenario, a strong opinion, a counterintuitive
fact, or direct address. See voice-persona.md for this client's example openers.

### The CTA
Every post ends with a CTA matched to topic and city, referencing the client's studio or
service area and using the booking link from profile §1. Never "I hope this was helpful!"
